Certificate and Key Renewal
REST Method 
POST
Description 
The Renew Certificate option should be used when a certificate has expired or is about to expire and you want to replace it with an updated certificate. This new certificate is based on the same private key and will have the same certificate ID.
Note: When replacing a key, its corresponding certificate must also be replaced.
Alteon Relative URI Path 
/config/sslcertimport?renew=1&id=abc&type=key&passphrase=test&src=file
id specifies the existing key/certificate name (both should be same)
type specifies the type: key/certificate
src specifies the source of the key/certificate: txt/file
Example
Request 
For a key:
 
https://10.82.41.135/config/sslcertimport?renew=1&id=WebManagementCert&type=key&src=txt&passphrase=admin
For a certificate:
 
https://10.82.41.135/config/sslcertimport?renew=1&id=WebManagementCert&type=cert&src=txt
Response 
On success:
 
The request has succeeded
On failure:
 
406 Not Acceptable.
 
Failed to extract Key.
406 Not Acceptable.
 
Failed to extract Certificate.
