Apply Alteon Configuration
HTTP Method 
POST
Description 
Applies pending configuration changes to the running configuration. All changes are applied immediately to the running configuration.
Alteon Relative URI Path 
/config?action=apply
Example 
Monitor the apply pending status with the following GET request:
Request 
 
https://172.17.151.225/monitor?prop=agApplyPending
Response 
 
{
"agApplyPending": 2
}
Value “2” means the apply is required.
Value “3” means no apply is required.
If required, perform the apply operation:
Request 
 
https://172.17.151.225/config?action=apply
Response 
Alteon responds with 200 OK to indicate that the apply request has started:
 
{
"status":"ok"
}
Monitor the status of the apply operation with the following GET request:
Request 
 
https://172.17.151.225/config?prop=agApplyConfig
Response 
 
{
"agApplyConfig": 1
}
Value “1” means no apply is in process, idle status.
Value “3” means the apply is in progress.
Value “4” means the apply is completed successfully.
Value “5” means the apply process failed.
Value “6” means the apply is completed successfully, with a warning.
 
Use the following GET request to check the apply errors and warnings table when the apply process ends, and the apply status is either failed (5), or completed (4) or (6):
Request 
 
https://172.17.151.225/config/AgApplyTable
Response 
 
{
      "AgApplyTable":[{
      "Index": 1,
      "StringVal": "Error: Enabled virtual server 11 has no IP address.U+0085Error: Apply not done.
Use 'diff' to see pending changes,U+0085 then use configuration
menus to correct errors.U+0085"
      }]
}
Note: Radware recommends that you check the apply table after every apply operation. Even when the apply operation is successfully completed, the table can hold warnings referring to the configuration changes that were performed. In most cases, these warnings recommend additional configuration changes.
