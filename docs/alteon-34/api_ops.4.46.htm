Identifying Scalar and Table Entities
The Alteon REST API Guide refers only to objects and parameters available in the Alteon Web Based Management interface. Alteon scalar and table entities are identified in the REST API using a shortened name based on the Alteon MIB name, so you can work on objects that are not available in the interface, such as the routing table.
MIB table names are changes as follows for REST API purposes:
Scalar entities use the same name as in the MIB table. For example, agNewCfgIdleCLITimeout.
Table entities use the MIB table name and change the first latter to upper case. For example: VlanNewCfgTable.
Table elements use a shortened form of the MIB table name, by truncating the common prefix from all table element MIB names. For example:
— Real server ID uses “Index” instead of slbNewCfgEnhRealServerIndex.
— IP address uses “IpAddr” instead of slbNewCfgEnhRealServerIpAddr.
