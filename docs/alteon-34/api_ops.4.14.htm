Save Alteon Configuration
HTTP Method 
POST
Description 
Saves applied configuration changes to memory. Changes that have not been saved to memory are lost after Alteon reboots.
After issuing the save command, the save status can be polled. The save failure table can be read in cases where the process ends with save failures.
Alteon Relative URI Path 
Issue the save command: /config?action=save (POST command)
Retrieve the save status: /config?prop=agSaveConfig (GET command)
— Response:
save(1) — Writes the current config block since the last save to flash memory, with backup.
idle(2) — There is no save operation in progress.
inprogress(3) —  The save operation is in progress.
complete(4) — The last save operation completed.
failed(5) — The last save operation failed.
If the save status fails, GET the failure reasons from the /config/AgSaveTable table.
Example
Request 
 
https://172.17.151.225/config?action=save
Response 
 
{
"status":"ok"
}
