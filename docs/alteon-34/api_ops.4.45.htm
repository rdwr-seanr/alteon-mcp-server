Generating Self-Signed Certificates
By default, the status of a self-signed server certificate is ungenerated. To make the certificate ready for use, send a PUT command to change the certificate status to generated, as follows:
To generate a self-signed server certificate 
1. Send an HTTP POST command to create a self-signed server certificate.
For example: user request URL https://<device IP>/config/SlbNewSslCfgCertsTable/abc/3/
where:
— “abc” represents the server certificate ID
— “3” represents the “server certificate” certificate type
which is "server certificate"
2. Set the parameters of the certificate in the request body.
For example:
 
{
"ID":"abc",
"CommonName":"mycommon",
"Type":"3",
"KeySize":"3",
"HashAlgo":"2",
"ValidityPeriod":"365"}
}
3. Change the certificate state to IDLE using a PUT command with the following body:
 
{
"Generate":"3"
}
where “3” represents the IDLE state.
4. Change the certificate state to GENERATE using a PUT command with the following body:
 
{
"Generate":"2"
}
where “2” represents the GENERATE state.
