Working with Augmented Tables
Some Alteon objects have many parameters. To send all object parameters without exceeding the maximum packet size, the tables describing such an object are divided into several augmented tables. For example, the real server table is divided into three parts, and the virtual service table is divided into six parts. Performing Operations on Augmented Tables describes how to work with augmented tables.
 
Performing Operations on Augmented Tables 
Operation
Required Action
Add a new object with augmented tables
Send the parameters of the first table using an HTTP POST command (this will create the object), then send augmented table parameters using an HTTP PUT command (to edit the existing object).
Edit parameters in one of the augmented tables
As for any other table.
Get all the values of the object
Request each augmented table separately.
Delete an object with augmented tables
Send the HTTP DELETE command with the name of the first table. For example, to delete a real server, use the SlbNewCfgEnhRealServerTable table name.
